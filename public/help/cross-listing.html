Cross-Listing

Publish your items to multiple marketplaces at once and keep everything in sync automatically.

Cross-listing means publishing a single inventory item as an active listing on multiple marketplaces simultaneously. Instead of logging into each platform and recreating the listing manually, VaultLister lets you create the item once — with photos, title, description, price, and condition — and then push it to any combination of connected platforms with a few clicks. All active listings share the same inventory, so when the item sells anywhere, the others are taken down automatically to prevent double-selling.
VaultLister currently supports cross-listing to Poshmark, eBay, Depop, Shopify, Facebook Marketplace, and Whatnot as live integrations. Mercari, Grailed, and Etsy are in active development and will be available in a future release — check the Roadmap for the latest timeline. Each supported platform has full listing creation, update, and delisting capability. Some platforms (eBay, Shopify, Depop) connect via OAuth API; others (Poshmark, Facebook Marketplace) use secure browser automation.
Every marketplace has unique required fields — eBay needs item specifics (brand, size, material), Poshmark requires a category and sub-category, Shopify needs product type and vendor. VaultLister maps your core listing data to each platform's schema automatically where possible. For fields that can't be auto-mapped, VaultLister surfaces a per-platform override panel in the cross-listing flow, so you can fill in platform-specific details before publishing. Required fields that are missing will be flagged before publish so nothing is rejected silently.
VaultLister monitors for sales via webhooks and periodic polling. When a sale is confirmed on any platform, VaultLister immediately marks the item as "Sold" in your inventory and queues delistings on every other platform where the item was active. For API-connected platforms, delistings typically complete within seconds. For automation-based platforms, the delisting is queued and processed within the next automation cycle. You'll see the sale appear in the Sales section with the platform, sale price, and timestamp.
Yes. In the cross-listing panel, each platform has an optional "Override" section where you can set a different price, title, or description for that platform only. For example, you might price a jacket at $45 on Depop but $55 on eBay to account for different fee structures or buyer demographics. Overrides are stored per-listing and applied each time VaultLister publishes or updates that listing on that specific platform. Your base inventory item remains unchanged.
VaultLister uses a combination of real-time webhooks (where supported by the platform) and scheduled polling to stay in sync. For eBay and Shopify, webhooks fire immediately on order creation, so sale detection is near-instant. For platforms without webhook support, VaultLister polls for new orders on a regular interval (typically every few minutes on paid plans). Listing status updates — such as price changes you make in VaultLister — are pushed to each platform's API within a few minutes of saving.
If a listing fails to publish on one platform, VaultLister records the error and shows it in the item's listing status panel. Common reasons include missing required fields (e.g., eBay item specifics), an expired OAuth token, or a platform-side API outage. VaultLister will not silently mark the listing as active if publishing failed — the status will show "Failed" with the error reason. You can review the issue, fix it (or re-authenticate if needed), and retry the publish from the same panel. See the Troubleshooting guide if errors persist.
