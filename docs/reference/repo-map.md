## 2. Key Commands

All commands run via `bun run <script>` unless noted.

| Purpose | Command |
|---------|---------|
| Start (production) | `bun run start` → `bun run src/backend/server.js` |
| Start (dev, with watcher) | `bun run dev` → bundles, watches bundle script, watches server |
| Start (background) | `bun run dev:bg` → `bun scripts/server-manager.js start` |
| Build frontend bundle | `bun run dev:bundle` → `bun scripts/build-dev-bundle.js` |
| Build (production) | `bun run build` → dev bundle + `scripts/build-frontend.js` |
| Run unit tests | `bun run test:unit` → `bun test src/tests/` (after test-bg setup) |
| Run security tests | `bun run test:security` → `bun test src/tests/security.test.js` |
| Run auth tests | `bun test src/tests/auth.test.js` (no dedicated script — call directly) |
| E2E smoke | `bun run test:e2e:smoke` → Playwright chromium, 6 smoke spec files |
| Syntax lint | `bun run lint:syntax` → `node --check` on `init.js`, `core-bundle.js`, `server.js` |
| Full lint | `bun run lint` → lint:syntax + lint:html + lint:css |
| Database init | `bun run db:init` → `bun run src/backend/db/init.js` |
| Database seed | `bun run db:seed` → `bun run src/backend/db/seed.js` |

## 7. API Route Registration

router imports, each named `<domain>Router`. They are registered after the middleware stack is

which routes are mounted and in what order.

## 9. Auth & Security Middleware

All middleware files live in `src/backend/middleware/`.

| File | Role |
|------|------|
| `auth.js` | JWT verification middleware; reads `JWT_SECRET` + `JWT_SECRET_OLD` from env; imports `jsonwebtoken`, `uuid`, `query`, `logger` |
| `csrf.js` | CSRF token middleware |
| `rateLimiter.js` | Rate limiting — required on all auth and public-facing routes |
| `securityHeaders.js` | CSP and security headers — changes require `bun test src/tests/security.test.js` |
| `cache.js` | Response caching |
| `cdn.js` | CDN-related headers |
| `compression.js` | Response compression |
| `errorHandler.js` | Centralized error handling |
| `featureFlags.js` | Feature flag middleware |
| `requestLogger.js` | Request logging |
| `validate.js` | Request validation |

Auth dependencies confirmed: `jsonwebtoken` (^9.0.2), `bcryptjs` (^2.4.3), `otplib` (^13.2.1),
`uuid` (^9.0.0).

CSRF is applied globally in `server.js` via `applyCSRFProtection()`. Do not add per-route CSRF
checks — use the global middleware.
